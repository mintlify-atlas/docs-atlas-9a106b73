---
title: "TaskQueue"
description: "Dependency-aware, event-driven task queue with topological resolution. Manages the full lifecycle of Task objects from creation through completion or failure."
---

## Overview

`TaskQueue` owns the mutable lifecycle of every task it holds. Tasks enter in `'pending'` state. When a task has unresolved dependencies the queue promotes it to `'blocked'` automatically; when those dependencies complete the queue restores it to `'pending'` and fires `'task:ready'` so orchestrators can react without polling.

```typescript
import { TaskQueue, createTask } from '@jackchen_me/open-multi-agent'

const queue = new TaskQueue()

queue.on('task:ready', (task) => scheduleExecution(task))
queue.on('all:complete', () => shutdown())

const task = createTask({ title: 'Research competitors', description: 'Identify the top 5 competitors' })
queue.add(task)
```

---

## `TaskStatus`

```typescript
type TaskStatus = 'pending' | 'in_progress' | 'completed' | 'failed' | 'blocked'
```

| Value | Description |
|---|---|
| `'pending'` | Ready to be picked up and executed. |
| `'in_progress'` | Currently being executed by an agent. |
| `'completed'` | Finished successfully; `result` holds the output. |
| `'failed'` | Execution failed; `result` holds the error message. |
| `'blocked'` | Waiting for one or more dependencies to complete. |

## `Task`

A discrete unit of work tracked by the orchestrator.

```typescript
interface Task {
  readonly id: string
  readonly title: string
  readonly description: string
  status: TaskStatus
  assignee?: string
  dependsOn?: readonly string[]
  result?: string
  readonly createdAt: Date
  updatedAt: Date
}
```

<ResponseField name="id" type="string" required>
  Unique identifier (UUID) generated by `createTask`.
</ResponseField>

<ResponseField name="title" type="string" required>
  Short human-readable name for the task.
</ResponseField>

<ResponseField name="description" type="string" required>
  Full description of the work to be done, typically used as the prompt or instruction for the assigned agent.
</ResponseField>

<ResponseField name="status" type="TaskStatus" required>
  Current lifecycle state of the task.
</ResponseField>

<ResponseField name="assignee" type="string">
  Name of the agent responsible for executing this task. When absent the task is unassigned and available for any agent.
</ResponseField>

<ResponseField name="dependsOn" type="string[]">
  IDs of tasks that must reach `'completed'` before this task can start. Tasks with unresolved dependencies are automatically set to `'blocked'`.
</ResponseField>

<ResponseField name="result" type="string">
  Output string set by `complete(id, result)`, or the error message set by `fail(id, error)`.
</ResponseField>

<ResponseField name="createdAt" type="Date" required>
  Timestamp set when the task was created. Never mutated after creation.
</ResponseField>

<ResponseField name="updatedAt" type="Date" required>
  Timestamp updated on every state change.
</ResponseField>

## `TaskQueueEvent`

Named event types emitted by `TaskQueue`.

```typescript
type TaskQueueEvent = 'task:ready' | 'task:complete' | 'task:failed' | 'all:complete'
```

| Event | Payload | Fired when |
|---|---|---|
| `'task:ready'` | `Task` | A task becomes `'pending'` and is available to execute (on `add` or when a dependency completes). |
| `'task:complete'` | `Task` | A task is marked `'completed'` via `complete()`. |
| `'task:failed'` | `Task` | A task is marked `'failed'` via `fail()`, or cascade-failed because a dependency failed. |
| `'all:complete'` | _(none)_ | Every task in the queue has reached a terminal state (`'completed'` or `'failed'`). |

## `createTask()`

Factory helper that creates a new `Task` with a generated UUID, `'pending'` status, and `createdAt`/`updatedAt` timestamps set to the current instant.

```typescript
function createTask(input: {
  title: string
  description: string
  assignee?: string
  dependsOn?: string[]
}): Task
```

<ParamField body="title" type="string" required>
  Short name for the task.
</ParamField>

<ParamField body="description" type="string" required>
  Full description of the work to perform.
</ParamField>

<ParamField body="assignee" type="string">
  Optional agent name to pre-assign this task to.
</ParamField>

<ParamField body="dependsOn" type="string[]">
  IDs of tasks that must complete before this task can run. The array is copied — mutations to the original have no effect.
</ParamField>

```typescript
const task = createTask({
  title: 'Research competitors',
  description: 'Identify the top 5 competitors and their pricing',
  assignee: 'researcher',
  dependsOn: [previousTaskId],
})
```

## `TaskQueue` class

### Constructor

```typescript
new TaskQueue()
```

Creates an empty queue with no tasks and no listeners.

---

### Methods

#### `add(task)`

```typescript
add(task: Task): void
```

Adds a single task. If the task has unresolved dependencies it is immediately set to `'blocked'`; otherwise it stays `'pending'` and `'task:ready'` fires.

---

#### `addBatch(tasks)`

```typescript
addBatch(tasks: Task[]): void
```

Adds multiple tasks at once. Dependency ordering is evaluated progressively as each task is added. Pre-sort with `getTaskDependencyOrder()` for predictable results when tasks within the batch depend on each other.

---

#### `update(taskId, patch)`

```typescript
update(
  taskId: string,
  update: Partial<Pick<Task, 'status' | 'result' | 'assignee'>>,
): Task
```

Applies a partial update to an existing task. Only `status`, `result`, and `assignee` fields are accepted. Use `complete()` and `fail()` for terminal state transitions.

<Note>Throws `Error` when `taskId` is not found in the queue.</Note>

---

#### `complete(taskId, result?)`

```typescript
complete(taskId: string, result?: string): Task
```

Marks a task as `'completed'`, records an optional result string, and automatically unblocks any dependents that are now ready to run.

Fires in order: `'task:complete'` → `'task:ready'` for each newly-unblocked task → `'all:complete'` if the queue is fully resolved.

<Note>Throws `Error` when `taskId` is not found in the queue.</Note>

---

#### `fail(taskId, error)`

```typescript
fail(taskId: string, error: string): Task
```

Marks a task as `'failed'` and stores `error` in the `result` field. Also cascade-fails every task that transitively depended on this one (preventing blocked tasks from remaining stuck indefinitely).

Fires `'task:failed'` for the failed task and for every cascade-failed downstream task.

<Note>Throws `Error` when `taskId` is not found in the queue.</Note>

---

#### `next(assignee?)`

```typescript
next(assignee?: string): Task | undefined
```

Returns the next `'pending'` task for the given `assignee`, or `undefined` if none exists. When `assignee` is omitted, behaves like `nextAvailable()`.

---

#### `nextAvailable()`

```typescript
nextAvailable(): Task | undefined
```

Returns the next `'pending'` task that has no `assignee` restriction, or the first `'pending'` task overall when all pending tasks have an assignee.

---

#### `list()`

```typescript
list(): Task[]
```

Returns a snapshot array of all tasks in the queue regardless of status.

---

#### `getByStatus(status)`

```typescript
getByStatus(status: TaskStatus): Task[]
```

Returns all tasks whose `status` matches the given value.

---

#### `isComplete()`

```typescript
isComplete(): boolean
```

Returns `true` when every task has reached a terminal state (`'completed'` or `'failed'`), or the queue is empty.

---

#### `getProgress()`

```typescript
getProgress(): {
  total: number
  completed: number
  failed: number
  inProgress: number
  pending: number
  blocked: number
}
```

Returns a progress snapshot with counts for each status bucket.

```typescript
const { completed, total } = queue.getProgress()
console.log(`${completed}/${total} tasks done`)
```

---

#### `on(event, handler)`

```typescript
on<E extends TaskQueueEvent>(
  event: E,
  handler: E extends 'all:complete' ? () => void : (task: Task) => void,
): () => void
```

Subscribes to a queue event. Returns an unsubscribe function; calling it is idempotent.

```typescript
const off = queue.on('task:ready', (task) => execute(task))
// later…
off()
```

## Helper functions

### `isTaskReady()`

```typescript
function isTaskReady(
  task: Task,
  allTasks: Task[],
  taskById?: Map<string, Task>,
): boolean
```

Returns `true` when a task can be started immediately — its status is `'pending'` and every task in `task.dependsOn` has status `'completed'`. Tasks with missing dependencies are treated as unresolvable and return `false`.

Pass `taskById` as a pre-built map when calling inside a loop to avoid rebuilding the map on each call (reduces complexity from O(n²) to O(n)).

---

### `getTaskDependencyOrder()`

```typescript
function getTaskDependencyOrder(tasks: Task[]): Task[]
```

Returns `tasks` sorted so each task appears after all of its dependencies (Kahn's topological sort algorithm). Tasks with no dependencies come first.

If the graph contains a cycle the function returns a partial result. Use `validateTaskDependencies()` to detect cycles before calling this in production paths.

```typescript
const ordered = getTaskDependencyOrder(tasks)
for (const task of ordered) {
  await run(task)
}
```

---

### `validateTaskDependencies()`

```typescript
function validateTaskDependencies(tasks: Task[]): {
  valid: boolean
  errors: string[]
}
```

Validates the dependency graph of a task collection. Checks for:

- References to unknown task IDs in `dependsOn`
- Self-dependencies (`task.dependsOn` includes its own `id`)
- Cycles (a task depending on itself directly or transitively)

Returns `{ valid: true }` when no issues are found, or `{ valid: false, errors: string[] }` describing each problem.

```typescript
const { valid, errors } = validateTaskDependencies(tasks)
if (!valid) throw new Error(errors.join('\n'))
```
